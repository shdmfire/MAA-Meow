# Roadmap

Planned features and improvements:

- [x] Native Root support
  In addition to the existing Shizuku approach, add a native Root mode to offer more ways to run the core, and to further improve device compatibility and usability.

- [x] Utility tools port
  Gradually port MAA-related utilities (mini-games, etc.) to cover more usable scenarios.

- [x] Scheduled tasks
  Start tasks automatically at preset times, reducing manual operations and better suiting daily idle farming and fixed routines.

- [x] Auto-update downloads
  Automatically check for and download app and resource updates on launch, with Mirror酱 mirror acceleration supported.

- [x] External notifications
  Push notifications through external channels when a task starts, ends, or encounters an error, so you can stay informed of the run status.

- [x] MaaFramework Controller migration
  Gradually migrate to the MaaFramework AndroidNativeController, improving the underlying implementation and laying the foundation for future features and maintenance.

- [ ] NCNN support for MaaCore
  Improve on-device runtime efficiency.
